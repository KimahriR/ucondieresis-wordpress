# Archive Index

## Purpose

This folder stores historical documentation, previous audits, old project snapshots, and reference material that is no longer part of the main living documentation.

Archive files are still useful for context, but they should not be treated as the current source of truth.

## Structure

### `audits/`

Technical audits and security reviews.

- `AUDIT_DEBUGGING_v1.0.4.md`
- `SECURITY_AUDIT_v1.0.3.md`

### `client/`

Client-facing or presentation-oriented historical materials.

- `DESCRIPCION_SITIO_ONEPAGE.md`
- `RESUMEN_EJECUTIVO_CLIENTE.md`

### `history/`

Project status snapshots tied to a specific point in time.

- `QUICK_STATUS.md`
- `STATUS_ACTUAL_7MAYO2026.md`

### `operations/`

Operational notes, cleanup plans, support references, and previous project summaries.

- `LIMPIEZA_PROYECTO.md`
- `RESUMEN_FINAL_CONFIG.md`
- `SOPORTE_TECNICO.md`

## Usage Rules

- Use archive files for history, traceability, and recovered context
- Do not rely on archive files as the main current documentation
- If an archived insight becomes important again, promote the distilled version into the core docs
